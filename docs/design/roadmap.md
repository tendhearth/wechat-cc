# Build Roadmap — 脊柱 + 情感陪伴 bundle

> **状态:P1→P4a 已全部实现并合入 master(PR #61, 2026-07-10)。** 剩余:Phase 4b(bundle 组装,等 dogfood 反馈)+ onboarding。原排序记录保留如下。日期:2026-07-09。
> 配套:`agent-social-network.md`(社交=倒刺)、`companion-liveness-layer.md`(活人感 + 设置=钥匙)。战略见 positioning 记忆。

## 排序三原则
1. **设置(钥匙)先行** —— 否则活人感功能不敢开(只能按最克制的最大公约数)。
2. **最便宜、信号最强的先做** —— 先验证"活人感"落地 + 建管道。
3. **主动类功能前必须有"校准引擎"** —— 否则就是骚扰(打扰是 per-user 偏好,不是统一克制)。

## 底层逻辑:能力是钩子,关系是倒刺
能力(码/分析)拉新但会商品化(低切换成本);关系(记忆+活人感+社交)留存(高切换成本=粘性)。→ **劲往倒刺上使。** 滩头(dev/自己)已验证离不开;情感陪伴段有真实用户在拉 = 近期 bet。

## Phase 0 — 已有地基(别重造)
_overview 合成(work+life)、observations/milestones、memory 工具、tier + 插件 admin-only、openai 便宜后端、**companion tick/push 主动调度器**、reply_voice。→ 后面多是**增强**,非新建。

## Phase 1 — 设置(钥匙)+ 最便宜的活人感
- **1a. per-user 偏好模型 + 对话式设置**(`/set` + 自然语言"少发点表情"/"多主动点"),存 per-chat。先只做对话式;临时网页(NAT 可达性坎)押后。
- **1b. 主动拆分**(多条短消息 + 类人节奏),受 1a 开关控制。
- 依赖:无 → **先做**。产出:立刻能 dogfood 的活人感 + 那把钥匙。

## Phase 2 — 校准引擎 + 第一个"主动关心"(情感陪伴的心脏)
- **2a. "该不该开口"校准引擎**:每人的 presence 量级(显式设置 + 从行为学)。所有主动行为过此门。增强现有 companion tick。
- **2b. 第一个主动关心**:memory/observations 驱动、有分寸的 check-in(先不做新闻——关心门槛低、差异化更高)。
- 依赖:1a。产出:情感陪伴核心,不骚扰;喂给已在拉的真实用户。

## Phase 3 — 更厚活人感 + 记忆喂养的个性化
- **3a. 表情包**(先确认 ilink 能不能发)。受设置控制。
- **3b. 分享信息/新闻**(个性化+精选+过校准门)= "打猎单机版",最难,放校准引擎稳了之后。
- **3c. 记忆→功能接线加厚**(兴趣/关系/情绪),**按功能按需加,别搞前置大工程**。
- 依赖:2a、记忆。

## Phase 4 — 人设 + 情感陪伴 bundle(组合前面所有)
- **4a. 人设/白纸养成/对标**:设置给种子 + 从记忆涌现。依赖 1a + 记忆。
- **4b. 组装 bundle** = 温暖人设 preset + 主动关心开高 + 相关插件 + 该段 onboarding + 云托管/便宜后端部署路径。
- 依赖:前面全部。这是**拼装**,非新建。

## 贯穿始终
- **每个 Phase 都要能 ship + dogfood**(你自己 + 情感陪伴用户),别全做完才上。
- 记忆加厚 = 持续、按需接线。
- 临时网页设置、云托管部署 = 押后(各有坎)。

## 起手式(从哪开始)
**Phase 1b「主动拆分」+ 一个开关**——最小、零依赖、立刻有体感、还顺手给设置层开第一个旋钮。几天内可 ship + dogfood。详见对话继续。
