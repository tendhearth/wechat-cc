# 活人感 / 陪伴层 — 设计笔记

> **状态:探索中(exploratory),尚未决定实现。** 思考记录,非实现规格。日期:2026-07-09。
> 主题:让 CC 感觉像一个**活的陪伴**,而不是一个工具——positioning 三根柱子里 **presence(在场)+ initiative(主动)+ identity(人格)** 那半,和"能干活"那半互补。

---

## 0. 为什么是一层(而不是零散功能)

单机护城河 = **记忆 × 在场 × 主动**。现在产品把"能干活 + 懂你"做得重,但**"在场感 / 活人感 / 人格"这层几乎没做**——CC 一说话就露馅(一堵文字墙、不发表情、从不主动、没有性格)。这一层就是把"关系"那半做厚。

四个功能各归一根柱子:

| 功能 | 柱子 |
|---|---|
| 表情包 · 主动拆分 | **presence(在场,像个活人)** |
| 分享时事/新闻 | **initiative(主动,不请自来)** |
| 白纸养成 / 人设 / 对标 | **identity(人格,和记忆复利)** |

---

## 1. 四个功能(是什么 / 为什么 / 关键张力)

**① 表情包设置** — presence,极微信原生。只发文字一眼假;会发表情包=活人感暴击,还是别人抄不了的微信主场。
- 张力:**品味 + 低打扰**(timing 不对很尬)、可配置(哪个表情库、什么人设配什么梗、频率)。
- 待确认技术点:**ilink 桥能不能发表情包/贴纸**——决定"现在能做"还是"要先打通发送通道"。

**② 主动拆分,有活人感** — presence。真人发微信是好几条短消息,不是一堵墙。bot 自己把回复拆 2–3 条、带类人节奏发。
- 张力:别过度拆(烦)、节奏(太快=机器人,太慢=卡)。

**③ 分享时事和新闻** — initiative,**但最危险**。这是"不请自来"那根柱子,也最容易做成没差异化的新闻推送 app。
- 差异化只在:**个性化(懂你兴趣)+ 精选(质量>数量)+ 时机对**。不是"念新闻",是"一个懂你的朋友甩你一句:欸这个你肯定感兴趣"。
- **连接**:这是社交网络那套 **"打猎/带明信片"的单机版**——agent 出去觅信息、只把好的带回。以后陌生层通了,新闻/八卦就是从 agent 网里"猎"回来的。可先做单机版,复用同一套"品味把关"。

**④ 白纸养成 / 人设 / 对标** — identity,和记忆复利。
- **白纸养成**:性格从你的互动里**涌现** → 独一无二、会复利(和记忆护城河同理),比固定模板更防抄。
- **对标的人**:告诉它想学谁 → 给"养成"一个种子人设起手。
- 和"保持**一个**一致的 CC、别碎片化人格"不冲突——这是塑造那**一个**身份,不是分裂它。

---

## 2. 钥匙:per-user 设置(整层的 enabler)

**核心洞见:不同用户容忍度不同,而 CC 现在"为了克制"什么都不敢开——正因为没有 per-user 调节,只能按"最克制的最大公约数"走。设置一旦有,活人感功能就能"想要的人开、怕烦的人关",不用一刀切猜。**

→ **设置不是附属功能,是解锁上面四个功能的前提。**

### 两条访问路径

**A. 对话式(`/set` 或自然语言)** —— 最贴品牌。
- "少发点表情" / "别推新闻了" / "多主动点" / `/set` → CC 当场调。
- 妙在:你不是"配置一个 app",是"**告诉你的朋友你的偏好**"——像真关系一样。这本身就是活人感。

**B. 临时网页(CC 给你一张一次性网页)** —— 适合"看得见比说得清"的设置。
- CC 生成一个**短时、带 token、scoped** 的链接;你在微信里点开,直接拖滑块(打扰频率、表情频率、新闻话题、人设/对标…),存回 daemon。
- 这其实是个**通用原语**:"**CC 给你一张临时网页**",用于一切"可视化比聊天强"的交互(设置 / 表单 / 选项对比 / 小看板)。不是常驻 web app、不要账号——用完即弃。

### 哲学:**合理默认 + 对话微调 + 可选深面板**
- 默认:**克制但不死**(有品味的低频活人感开着)。
- 对话:随口一句就调("别烦我" / "多发表情")。
- 深面板:power user 想要滑块时,才给那张临时网页。

### 权限 scope(接 tier 模型)
- 非 admin 的自助设置主要是"**你打扰我多少**"(静音 / 频率),不是重配机主的 CC。
- admin 全量。

---

## 3. 必须守的战略护栏

这四个都往"**陪伴**"极推。positioning 早破过:身份在"开发者远程工具"和"AI 陪伴"之间发虚,而且**别在"谁聊得好"上跟 Character.ai/Replika 拼**(他们主场,必输)。

> **活人感/人设是"能干活 + 懂你 + 在微信"之上的一层皮,不是替代品。** 一个很会卖萌、干不了活的 bot = 又一个 Character.ai(输)。魔法在于:一个**能真干活、还懂你、还有活人感、还是你自己的** agent。**底座别丢。**

---

## 4. 贯穿所有的一条生死线:低打扰

positioning 北极星:"必须 **earn** presence、知道何时闭嘴、**别变成任务提醒**"。
- 表情、拆分、新闻、主动——每一个做过头都从"惊喜"变"骚扰",整层魔法就破。
- **per-user 设置(第 2 节)正是低打扰的调节旋钮**——它让"多少算过头"由每个用户自己定,而不是我们猜。

---

## 5. 待想清楚的技术/开放问题

- **ilink 能否发表情包/贴纸**(能力边界,决定 ①)。
- 拆分节奏怎么自然(②)。
- 时事新闻的个性化+精选+时机;和"打猎"复用品味把关(③)。
- **人设别碎片化**,保持一个一致 CC(④)。
- **临时网页的可达性**:daemon 在 NAT 后 / 常离线,手机(微信)怎么打开?——要么局域网/隧道,要么走 desktop 本地服务,要么一个内容不可见的中继(和社交网络那边的 relay 是同一类难题)。安全:链接必须 **tokenized + 短时 + scoped**(只本人能开、只改自己的设置),接 tier/token 体系。
