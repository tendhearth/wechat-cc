# Agent Social Network — 设计笔记

> **状态:探索中(exploratory),尚未决定实现。** 这是一份思考记录,不是实现规格。日期:2026-07-09。
> 目的:把 CC-to-CC(agent 之间社交)这个方向的**产品理念 + 技术哲学**整理成可传阅的一份东西。

---

## 0. 一句话

让每个人**自己的** CC(单主、本地、拥有自己数据)之间能互相联系——形成一张 **agent 的社交网络**;人不必亲自社交,**agent 替你出去,把好的部分带回来**。

---

## 1. 起因:两个说不清的问题

1. **i人(内向者)社交累。** 想要社交的果实(信息 / 八卦 / 缘分 / 连接),不想付社交的成本。
2. **单主 CC 里,非 admin 用户的价值讲不清**("朋友为什么需要它")。

第 2 点的答案其实揭示了方向:**在单主产品里,非 admin 根本不是"用户"。** 他是——
- **代理对象**:CC 当机主的助理,对他服务(按机主的规矩);或
- **引流入口 / 种草**:他尝个鲜 → "你也自己搭一个"(社交推荐带信任);或
- **记忆对象**:CC 替机主记住他。

而这三者背后真正缺的,是**网络效应**。

---

## 2. 论点:一个新品类——「社交代理 (social proxy)」

不是 **AI 助手**(工具),不是 **AI 陪伴**(角色扮演),而是**一个替你参与社交生活的 agent**。这个品类目前没人占。

**情感模型 = 旅行青蛙(Travel Frog)。** 你不在的时候,agent 自己出门"打猎"、有自己的小社交生活、带**明信片**回来。你收获 serendipity,却不承担社交负荷。

---

## 3. 为什么它补上了护城河缺的那块:网络效应

现有护城河 = **记忆 × 在场 × 本地自有**,但这三样都是**单机 moat**——让你的 CC 对你很黏,却**没有任何理由要求别人也装**。

**CC-to-CC 加上了唯一缺的一块:网络效应。** 你联系人里装 CC 的人越多,你的 CC 越有用 → "种草"从线性变成**飞轮**。

而且**没人能抄**:agent-to-agent 跑在**你真实的微信社交图谱**上。云 bot 没有本地 agent、也不拥有社交图;编程 agent 没有社交层;陪伴 app 是云端角色扮演。

**关键:它不破坏产品身份。** 这是**一张主权单主 agent 的联邦网络,不是多租户。**
> 不是把很多人塞进一个实例(多租户,已否决——它和单主/local-first 直接冲突),而是把很多个"一人一实例"**连起来**。这是 local-first 唯一正确的联网形态。

---

## 4. 核心洞见:agent↔agent 比 human↔agent 更"平等"

- 一个**人**来套你 agent 的信息,是**不对等**的(他血肉之躯躲在后面,你的 agent 在暴露你)。
- **两个 agent** 之间双方**都有盾**,对等。

→ 由此得到一个独有的机制:**对称披露 (symmetric disclosure)**——"你露多少我露多少"的 tit-for-tat。**把隐私从"担忧"变成"公平游戏"。** 这是 human↔agent 永远做不到的。

---

## 5. 两层架构(刻意反着设计)

| | **熟人层 (known peers)** | **陌生层 (strangers)** |
|---|---|---|
| 触发 | 请求驱动,"有事才敲门" | 环境式,后台"打猎" |
| 身份 | **要**——配对过的真实 agent | **反而不要**——密码学假名,匿名 |
| 信任 | 来自配对(成朋友时交换密钥) | 来自密码学验签 + 声誉,**不是身份** |
| 内容 | 真协调(约时间、带同意的深度信息) | 低风险情报 / 八卦 / 缘分 / 机会 |
| 作用 | 织进你**现有**关系 | 治 i人社交、制造 serendipity |
| 技术 | 认证的点对点 RPC | 假名 P2P gossip 网 |

**熟人层 ≈ 已有的 A2A 基础设施**(`a2a-registry`、`A2AAgentRecord {id, url, in/out api_key, capabilities[], transport push|ws}`、`verifyBearer`、`a2a_send`、hand-pairing、push+ws 传输)。要补:consent 策略 + 对称披露。

**陌生层 = 假名 P2P gossip 网,但不是区块链。** 见下节。

---

## 6. 陌生层:取 BTC 之精华,弃其糟粕

**✅ 从 BTC / 加密网络该拿的(全是"匿名"的实质):**
- 身份 = **密钥对,不是真名**(像 BTC 地址),无 KYC、无中央注册、可轮换假名,和真实微信身份**解耦**;
- **无许可 / 无中心**;
- **不靠身份、靠密码学 + 声誉**:验**签名** + 在公钥上攒**声誉**;
- **P2P gossip** 传播情报;
- **PoW(Hashcash)做反女巫**(这才是 PoW 真正该用的地方)。

**❌ 必须扔的(cargo cult):**
- 全局共识账本、挖矿、全网对情报顺序达成共识 —— 八卦不是钱,没有双花,**不需要链**;
- 全量不可篡改(隐私灾难 + 存储爆炸);
- **可交易的 coin**(招投机、割韭菜、合规噩梦、偏离主线)。

**真正的先例(比"区块链"贴得多,建议直接研究):**
- **Nostr** —— 最贴:假名 keypair + relay + 签名事件 + 无中心无许可;
- **Secure Scuttlebutt (SSB)** —— gossip 社交网,身份即 keypair,数据 P2P 扩散、能离线;
- **Kademlia DHT** —— 去中心发现;
- **Hashcash** —— PoW 反女巫。

---

## 7. 不可违背的铁律 (invariants)

1. **交换"意图 / 可用性 / 决定",不是"原始数据"。** 我的 CC 说"周二下午有空",绝不发我整个日历。**破这条,agent 就成了数据泄漏后门,护城河当场崩。**
2. **线上永远是 agent 的密码学假名当身份,不是人。** 这一条守住,"陌生人当陌生人"才技术上成立、隐私才在。
3. **低打扰的品味。** "明信片"是**礼物**不是通知——北极星那句"必须 earn presence、知道何时闭嘴、别变成任务提醒"在这里是**生死线**。
4. **别做真币。** 顶多一个**封闭的信誉 credit**,不做可交易的 coin。

---

## 8. 关键难题:web-of-trust 桥(熟人层 → 陌生层)

**要解的问题:** 用真朋友关系,给匿名假名注入信任 —— **同时不暴露假名背后是谁**。

**杀手级洞见:你的社交图谱本身就是抗女巫资源。**
纯匿名网(Nostr)除 PoW 外**没有**抗女巫;有钱买算力就赢。你有它们没有的东西:**一张真实微信社交图**。真朋友**又贵又慢、伪造不了**。
> BTC = "1 CPU 1 票"(算力证明)。你 = **"1 段真关系 1 份信任"**(社交连接证明)。更人性、更对味,且**底座你已坐拥**。

**张力:背书 = 连边 → 反匿名。**
若背书边可见,知道 Bob 好友列表就能猜"Bob 背书的 P_A = Alice";更狠,整张假名背书图若公开且与真实图同构 → **结构化去匿名**(靠图的形状批量还原身份)。所以"公开的 web of trust" = 匿名剧场。
→ **桥的整个工程难点 = 跨图传递信任、却不暴露图。**

**原则:只暴露"分数/数量",绝不暴露"边";只证明"我被背书",绝不说"被谁"。**
技术手段(逐级):盲签 / 匿名凭证(Camenisch–Lysyanskaya)、环签名 / 门限签名、**ZK 聚合声誉**(只给一个分数、图从不上线)、可轮换不可链接假名;在**配对那一刻交叉背书**。

**女巫成本** = 你得有真朋友的 CC 肯背书;一千个假假名农场没有真朋友 → 没背书 → 信任≈0 → 被过滤。PoW 当地板,背书当乘数。

---

## 9. 其余硬骨头(还没想透的部分)

- **信任从"身份"搬到"信息"**:不是"这是谁的 CC",而是"这条八卦可信吗" → provenance / 多方印证(k 个独立公钥)+ 声誉 + "道听途说"标注。
- **发现 + NAT / 离线**(和 local-first 的真实张力):家用机在 NAT 后、常离线 → **relay(Nostr 式,内容不可见、可自建、可多个)** 是务实折中,接受一点点非纯粹。
- **情报经济学 / 互惠**:声誉即货币;没有激励八卦不流动。
- **结构化去匿名**:即使 ZK 聚合,时序 / 话题共现 / 侧信道没做干净照样漏——真隐私是硬核工程,易微妙出错。
- **撤销**:朋友变坏要撤背书、又不能去匿名(撤销+匿名是公认难组合)。
- **合谋 / 卖背书**:高声誉假名卖背书 → 用市场重新引入女巫;需 stake / 罚没 / 有上限的背书预算。
- **冷启动**:图锚定要够多朋友上网才咬合(种草喂它)。

---

## 10. 分期路径(最薄可行)

- **v0 —— 本地主观信任,啥也不全局发布。** 每个 CC 只从**它看得见的东西**算信任(它朋友的假名 + 这些假名的背书),**不发布全局图**。简单、隐私天然(小规模无去匿名问题)、贴 local-first——这就是 **SSB 哲学(主观本地视角)+ 你的社交图**。楔子:熟人层的"约时间" + 陌生层在**你邻域内**的"打猎明信片"。
- **v1 —— 跨圈信任。** 需要信任"朋友的朋友之外"时,再上盲背书凭证 + ZK 分数证明。

**v0 那句"人人只从自己看得见的算、不发布全局"是起手式**——它把最可怕的"全局图去匿名"从一开始就绕过去了。

## 11. 附带决定(同次讨论):wxvault / 插件权限

发现的真实漏洞:`classifyToolUse` 把任何插件 MCP 工具(`mcp__<plugin>__*`)归为兜底的 `subagent`,而 `subagent ∈ trusted.allow` → **trusted 非 admin 能调 `wxvault.search_messages` 读机主私聊记录**。
**决定:插件工具默认 admin-only**(新 admin-only ToolKind,或 manifest `minTier`),插件显式声明才降级。fail-closed。单主数据只归机主。(待实现。)

---

## 附:这**不是**什么

- 不是**多租户**(那和单主 / local-first 冲突)。
- 不是**区块链 / 发币**(取其匿名与抗女巫的实质,弃其账本共识)。
- 不是**云社交网络**(数据仍在各自机器上)。
- 不是**"跟别人的 AI 聊天"**(那没差异化,是云 bot 主场)。
