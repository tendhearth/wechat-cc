# Persona (白纸养成) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Global self-cultivated `persona.md` (owner chat's memory dir) injected into every chat's prompt; cultivation guidance shown only in the owner chat.

**Architecture:** Two prompt sections (`personaSection(content)` + `personaCultivationSection()`) gated by `BuildSystemPromptArgs.persona?: string` / `personaCultivate?: boolean`; one bootstrap thunk `personaFor?`; main.ts wires fresh-per-spawn file read + owner-only cultivate flag. Zero new tools/routes/stores.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-10-persona-design.md`. Cap injected content at 4000 chars (slice + note); byte-identical prompt when persona absent/empty AND cultivate false/absent; cultivation guidance ONLY when cultivate; absent `personaFor` dep ⇒ both omitted.
- TDD; tsc clean; explicit `git add`; unrelated WIP may exist in tree.

### Task 1: prompt sections + args

**Files:** `src/core/prompt-builder.ts` (+test)

- `export function personaSection(content: string): string` — heading `## 你的人设(persona)`, then: 下面是你自己维护的人设档案(persona.md),这就是你的性格和说话方式,所有对话里保持一致地做这个"人": + the content (sliced to 4000 chars).
- `export function personaCultivationSection(): string` — heading `## 人设养成(persona.md)`(owner chat only), when-to-use framing: persona.md 在你和主人的对话记忆里,是白纸养成的性格档案;随着相处把逐渐形成的说话风格、性格特质、主人的调教(对标谁/语气/雷区)写进去(memory_write);简短、条目化;**改动要克制——人格是缓慢生长的,不是每天重写**;主人说"对标 XXX / 以后这样说话"时,更新并口头确认。
- `BuildSystemPromptArgs` gains `persona?: string` + `personaCultivate?: boolean`; buildSystemPrompt appends `args.persona && args.persona.trim().length > 0 ? personaSection(args.persona) : ''` and `args.personaCultivate ? personaCultivationSection() : ''` (place persona right after baseChannelSection — identity comes first; cultivation next to careSection).
- Tests: section contents (人设/一致/4000-cap test with a 5000-char string asserting `.length` bounded); strict `toBe` byte-identical when persona absent AND when `''`/whitespace; cultivate gating both ways.

### Task 2: bootstrap thunk + main wiring + full verification

**Files:** `src/daemon/bootstrap/index.ts`, `src/daemon/main.ts` (+ touched tests)

- `BootstrapDeps.personaFor?: (chatId: string) => { content?: string; cultivate?: boolean }` (doc like careLevelFor); buildInstructions impl: `const p = deps.personaFor?.(chatId); ... persona: p?.content, personaCultivate: p?.cultivate === true`.
- main.ts (next to careLevelFor): wire
```ts
personaFor: (c) => {
  const ownerChat = loadCompanionConfig(stateDir).default_chat_id
  if (!ownerChat) return {}
  const fs = makeMemoryFS({ rootDir: join(stateDir, 'memory', ownerChat) })
  return { content: fs.read('persona.md') ?? undefined, cultivate: c === ownerChat }
},
```
(verify `makeMemoryFS`'s actual import path + `read` signature — `src/daemon/memory/fs-api.ts`; fresh read per spawn = hot-editable, cheap. If makeMemoryFS construction per call is heavy, hoist the rootDir resolution but keep the read fresh.)
- Add a bootstrap.test.ts case: personaFor returning content ⇒ instructions contain the persona content; cultivate true only for owner chatId; absent dep ⇒ byte-identical (reuse care-test fixture style).
- FULL verification: tsc clean; full suite green (git-stash triage); e2e green (harness has no persona.md ⇒ inert).
- Commit: `feat(persona): wire owner persona.md into every chat's prompt`.
