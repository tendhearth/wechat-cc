# v0.5.13 — memory section: AI's mental model, not a database

**Date**: 2026-05-07
**Tag**: `v0.5.13` + `desktop-v0.5.13`
**Scope**: `memorySection()` rewrite in `prompt-builder.ts`. **No code logic changes** — purely the system prompt fed to claude / codex sessions on first dispatch.
**Predecessor**: [v0.5.12 — codex sessions render in dashboard 会话 tab](2026-05-07-v0.5.12.md)

---

## Why this exists

Through v0.5.12 the memory prompt told the agent things like:

> 你有 `~/.claude/channels/wechat/memory/` 目录，完全由你自治。组织你自己定。

And let the agent freely create files. After ~2 weeks of dogfooding the maintainer's `o9cq800sOb...@im.wechat` chat had this layout:

```
memory/<chat_id>/
├── profile.md                      ← user-identity (durable)
├── infra-llm-bridge.md             ← topic notes
├── agent-cli-landscape-2026.md     ← topic notes (with date in filename)
├── session-handoff-20260426.md     ← dated handoff
├── project-hearth.md               ← project notes (already prefixed!)
└── wechat-cc-restart-pattern.md    ← gotcha / pattern
```

Not catastrophic at 6 files, but it would collapse at 30+. The prompt didn't give shape. The agent's natural use of prefixes (`project-`, `session-handoff-`) hinted at the categories it WANTED but had no taxonomy for.

A second, deeper realisation came from a design conversation with the maintainer: **memory should be the AI's mental model of the user, not a structured fact database**. Sessions (jsonl) already provide the verbatim record. Memory should be selective, narrative, biased, evolving — like a friend's memory of you, not a third-party monitoring snapshot.

## What v0.5.13 changes

`memorySection()` in `prompt-builder.ts` is rewritten end-to-end. New framing:

1. **Memory ≠ database. Memory = your mental model.** Allowed to be wrong, allowed to evolve, allowed to have interpretation. Sessions are the honest record; memory doesn't need to be.

2. **Three file types only**:
   ```
   memory/<chat_id>/
   ├── profile.md          # who they are, what they're doing (durable, edit-in-place)
   ├── preferences.md      # how to work with them (durable)
   └── notes/<slug>.md     # everything else, AI's natural slug choice
   ```
   No `projects/`, no `topics/`, no `handoffs/`, no `inbox/`. A `notes/compass.md` can hold both architecture decisions AND the user's emotional take on the project — that's integration, the way a friend remembers things.

3. **First-person, not third-person**:
   > "他这周在试验新流程，我感觉他比较在意 X." ← yes
   >
   > "用户表现出对 X 的关注。" ← no

4. **Proactive surfacing as a behaviour, not a storage type**:
   > 扫 notes 时如果看到很久没碰、跟当前话题相关的内容，自然地织进回复："你上个月提过 Y，跟现在这个有联系吗？" 这是朋友式的"想起来了"，不是 todo 系统的催促。
   
   Same mechanism handles "P1 user wants forgotten-idea catch-up" and "P2 user wants emotional check-in" — the difference is what content is relevant to surface, not how it's stored.

5. **Persona influences reading lens, not file structure**:
   - `小助手`: prioritises unfinished ideas, decision points, follow-ups
   - `陪伴`: prioritises emotional continuity, life-context, mood
   
   Same `notes/*.md` files. Different reading angle.

## What stays unchanged

- Memory tools (`memory_read` / `memory_write` / `memory_list`) — unchanged.
- Existing memory files — NOT migrated. The AI naturally consolidates over time as it reads/writes (the prompt explicitly endorses "睡觉时归并").
- Companion observations / events / milestones streams — still auto-generated, still raw signals. The new framing reclassifies them as **inputs to memory writing**, not memory itself, but no code change needed.
- Dashboard 记忆 pane — already shows .md files; will reflect the new structure organically as the AI starts using `notes/`.

## Why no code changes

Nothing about the file system, the cli `memory` commands, the dashboard, or the daemon needs to move. This is purely a behaviour change driven through the system prompt. The first dispatch in each session injects the new prompt; the AI sees it; subsequent memory_writes use the new convention.

That's also why the test suite is unchanged at 1316 passing — there's nothing new to assert on. (The prompt-builder tests just check sections exist; they pass because we didn't remove the section.)

## What you'll see

Next time you /chat or /cc and the agent updates memory, you should see:

- Existing root files **not touched** (the AI is allowed to leave legacy alone).
- New writes go into `notes/<slug>.md` instead of root.
- Over time, the AI consolidates: e.g., merging `infra-llm-bridge.md` + `agent-cli-landscape-2026.md` into a unified `notes/agent-cli-landscape.md`, or rolling `wechat-cc-restart-pattern.md` into `notes/wechat-cc.md`.
- First-person tone in the notes (not "user observed X" — "X 这事他纠结了好几天").
- Occasional natural callbacks: "对了，你上个月提到 Y，跟今天这个相关吗？" That's the AI scanning notes/ before replying.
