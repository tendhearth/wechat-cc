## Companion — inline quoted content

When a user quotes/replies to an earlier message, the daemon now surfaces the **full** quoted text to the agent as a structured `<quote type="…">…</quote>` element, instead of the old truncated `[引用: …]` snippet and a meaningless `quote_to` id.

- ilink delivers the quoted message inline in `ref_msg` (it carries **no** stable id, and there is no message store), so the right design is to surface that content, not build a lookup tool. `parseUpdates` extracts the richest available text (`message_item.text_item.text` → `unsupported_item.text` → `title`) plus a type label, and `formatInbound` renders it as `<quote type="text|image|voice|file|…">原文</quote>` at the **start** of the message body so the agent reads the context before the user's new line.
- The dead `quote_to="<id>"` attribute is gone; the system prompt now documents the `<quote>` element to the agent.
- A degenerate `ref_msg` with neither a known type nor any text is skipped (no empty `<quote>`).
- Scope note: this deliberately ships **no** message store / lookup tool — the original "per-turn msg_id + turn-by-id resolution" backlog framing was infeasible (ilink gives no quoted-message id). General chat scrollback remains a separate, larger, unstarted feature.
